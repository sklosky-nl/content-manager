ðŸ“– Using URL Query Parameters

You can customize views and filter data by adding query parameters to the /display URL. Multiple parameters can be combined using &.

Available Parameters

Parameter Values Description Example
range today, tomorrow, this-week, next-week, next-10-days, this-month, next-month Filter by predefined date range ?range=today
date YYYY-MM-DD or MM/DD/YYYY Show events for a specific date ?date=2026-01-15
start YYYY-MM-DD or MM/DD/YYYY Start date for custom range (use with end) ?start=2026-01-01&end=2026-01-31
end YYYY-MM-DD or MM/DD/YYYY End date for custom range (use with start) ?start=2026-01-01&end=2026-01-31
location or space Location name (partial match) Filter by location/space name ?location=Events Bay
source all, wild-apricot, skedda Filter by data source ?source=skedda
format desktop, mobile, kiosk, print Display format/layout ?format=kiosk

Example URLs

{{ base_url }}/display?range=next-week&format=desktop
{{ base_url }}/display?range=next-10-days&format=kiosk
{{ base_url }}/display?date=2026-01-15&location=Events Bay
{{ base_url }}/display?start=2026-01-01&end=2026-01-31&source=skedda&format=kiosk
{{ base_url }}/display?range=today&format=mobile&source=wild-apricot

Tips

  • Parameters are case-insensitive (except for date formats)
  • You can combine multiple parameters: ?range=today&format=kiosk&source=all
  • Location/space filtering uses partial matching (e.g., "Events" matches "Events Bay")
  • If no date parameters are provided, all future events are shown by default
  • The QR code on each display page shows the current URL with all parameters
